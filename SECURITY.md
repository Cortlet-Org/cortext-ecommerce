# Security Policy

## Supported Versions

Cortext is actively maintained on the `main` branch.

Security fixes are applied to the latest version only.

---

## Reporting a Vulnerability

If you discover a security vulnerability in Cortext, please **do not** open a public GitHub issue.

Instead, report it privately via email:

support@cortlet.com

Please include **"Cortext Security Disclosure"** in the subject line.

---

## What to Include

When reporting a vulnerability, please provide:

- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Suggested remediation (if known)
- Relevant logs or screenshots

Providing detailed information helps us assess and resolve issues efficiently.

---

## Disclosure Process

All security reports will be:

- Reviewed confidentially
- Investigated promptly
- Addressed based on severity

We appreciate responsible disclosure and will work to resolve confirmed vulnerabilities in a timely manner.

---

Thank you for helping keep Cortext secure.
