## Cortext — A Zero-Assumption Ecommerce Foundation

Cortext is a lightweight ecommerce demo platform built to demonstrate clean, vendor-agnostic architecture.

It provides a minimal foundation for building ecommerce applications without enforcing:

- A specific payment provider
- A specific cart implementation
- A specific backend
- A specific data model

You bring your own integrations.  
You control the architecture.

---

## Tech Stack

- Next.js (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui (with Radix primitives)
- next-themes

Intentionally minimal.  
No forced integrations.  
No baked-in payment providers.  
No hidden coupling.

---

## Features
- Light, Dark, and System themes support (via `next-themes`)
- Minimal, black-white modern UI
- Inter font (via `next/font`)
- Runnable, usable template.
---

## The Problem

Many ecommerce starter projects ship with:

- Pre-wired integrations (e.g. Stripe, Shopify)
- Opinionated backend assumptions
- Large plugin ecosystems
- Excess boilerplate

While convenient, these assumptions can introduce tight coupling, making it harder to replace providers or restructure systems later.

---

## Design Principles

- Minimal surface area
- Explicit over implicit
- Composition over coupling
- Infrastructure neutrality
- Clean separation of concerns

## Usage

### Run Locally

To run locally, you must clone the project, change directory, install and run dependencies.
Dependencies are managed with NPM.
```bash
git clone https://github.com/Cortlet-Org/cortext-ecommerce.git
cd cortext-ecommerce
npm install
npm run dev
```

Open:

http://localhost:3000
